# Coding Rules

Priorities:
1. Stability over cleverness
2. Minimal disruption
3. Preserve backward compatibility
4. Avoid dependency bloat
5. Avoid token-heavy logic
6. Keep VSCode-native UX

Style:
- Prefer explicit TypeScript
- Prefer small isolated modules
- Avoid hidden side effects
- Avoid giant abstractions
- Keep agent behavior deterministic
